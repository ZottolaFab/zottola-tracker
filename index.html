🔗 Projects & Work Types

Zottola Shop Tracker - Projects & Work Types

👥 Employees — Sync Setup Required

Employees are set up to sync from the Google Sheet the same way Projects and Work Types do, but this needs a one-time setup that isn't done yet:

1. Add an "Employees" tab to the Sheet (same format as the Projects/WorkTypes tabs — a header row, then one name per row in column A)
2. Add a getEmployees action to the Apps Script proxy that reads that tab, matching the existing getProjects/getWorkTypes pattern
3. Redeploy the Apps Script (Deploy → Manage deployments → Edit → New Version → Deploy)

Until that's done, the app safely falls back to its built-in employee list plus anything added below as temporary — nothing breaks, it just won't be pulling from the Sheet yet.

➕ Add Temporary (This Device Only)

Need to log hours for someone or something not yet in the Google Sheet? Add it here — it works immediately on this device, but it will not sync to other tablets or to the Sheet. Once it's added to the Sheet for real, remove the temporary version below so the list stays clean.

🔵 Prevailing Wage Projects

Prevailing wage projects are managed through the Google Sheet.

To mark which projects use prevailing wage rates:
1. Go to the Google Sheet
2. Open the "PWProjects" tab
3. Add project names in column A (one per row)
4. Save the sheet
5. Next time you open the app, PW projects will sync automatically

✓ When a PW project is selected on Track tab, a 🔵 PW PROJECT badge appears and onsite hours automatically apply prevailing wage rate.
